まいどっ！

この間自宅サーバ（Ｆｅｄｏｒａ１０）にＷｅｂカメラを構築したんですが、
[about WAKITAs' server](http://wakita.no-ip.com/server/index.html "about WAKITAs' server") にも時々Ｗｅｂカメラがダウンするとかの記載があって心配していたんですが・・・

とうとう、うちのもダウンしました。

現象は[WAKITA](http://wakita.no-ip.com/server/index.html "WAKITA")さんと同じかどうかはあんまり詳しくないのでわかりませんが、[WAKITA](http://wakita.no-ip.com/server/index.html "WAKITA")さんがされているような対応を真似させてもらいました。
（ [こちら](http://wakita.no-ip.com/server/respawnd/index.html "こちら")を参照 ）

### 作業手順

#### １．respawnd なるファイルを[こちら](http://wakita.no-ip.com/server/respawnd/download/respawnd.tar.gz "こちら")からダウンロード

``` text
# wget http://wakita.no-ip.com/server/respawnd/download/respawnd.tar.gz

```

#### ２．ダウンロードしたファイルを展開・ビルド・インストール

``` text
# tar zxvf respawnd.tar.gz
# cd respawnd-1.0.0
# make
# make install

```

#### ３．/etc/rc.d/rc.local の webcam 起動の部分を以下のように変更

    （ フォルダ構成は当方のケース ）

``` text
# vi /etc/rc.d/rc.local
--------------------------------------------------
usr/local/bin/webcam /usr/local/etc/webcamrc > /dev/null 2> /dev/null &
        ↓
<font color="green">#</font>/usr/local/bin/webcam /usr/local/etc/webcamrc > /dev/null 2> /dev/null &
<font color="green">/usr/local/bin/respawnd /usr/local/bin/webcam /etc/local/etc/webcamrc</font>
--------------------------------------------------

```

---

以上で完了のはず。

とりあえず当分はこれで運用してみて、[about WAKITAs' server](http://wakita.no-ip.com/server/index.html "about WAKITAs' server") のページにもあるように
respawnd による syslog の出力 （ /var/log/messages ）
をチェックしてみることにします。

結果は、また後日。。。
