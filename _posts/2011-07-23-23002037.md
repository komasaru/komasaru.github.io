今日はJavaScriptのライブラリの「jQuery」を「Ruby on Rails」環境に導入してみました。
今までは「jQuery」の必要性は感じませんでしたが、このたび使用したいJavaScriptで「jQuery」が必要だったので。

<!--more-->

以下が作業記録です。

### 作業記録

#### 前提条件

- 使用OSはWindowsXP ( Linuxは Scientific Linux 6.0 )
- Rubyのバージョンは 1.9.2-p180
- Railsのバージョンは 3.0.5
- WebサーバはRails標準の Mongrel ( Linuxは Apache + Phusion Passenger )
- HTML、Railsの若干の知識がある。

※Rails3.1系やRails2系は若干操作が異なるので、「[rails/jquery-ujs - GitHub](https://github.com/rails/jquery-ujs "rails/jquery-ujs - GitHub")」を参照すること。
※Linux版でもほぼ同様の手順で可能。（ScientificLinux6.0 + Ruby1.9.2-p180 + Rails3.0.3 で確認）

#### １．「Gemfile」の編集

「[rails/jquery-ujs - GitHub](https://github.com/rails/jquery-ujs "rails/jquery-ujs - GitHub")」の説明にあるとおり、Gemfileに以下の記述を追加する。

``` text
gem 'jquery-rails', '>= 1.0.12'

```

#### ２．「jQuery」のインストール

コマンドプロンプトでRailsアプリのディレクトリへ移動し以下のようにしてインストールする。

``` text
bundle install

```

以下のようにエラーが発生する場合、

``` text
uninitialized constant Gem::SilentUI (NameError)

```

これは「bundle」のバージョンが古いからのようです。
この場合は次のようにアップデートしてから再度「bundle install」する。

``` text
gem update bundler

```

#### ３．JavaScriptの整理

以下のコマンドで既存の「public/javascript」配下のjavascriptと「jQuery」に含まれているものの重複をチェックし整理します。

``` text
rails generate jquery:install
      remove  public/javascripts/prototype.js
      remove  public/javascripts/effects.js
      remove  public/javascripts/dragdrop.js
      remove  public/javascripts/controls.js
     copying  jQuery (1.6.1)
      create  public/javascripts/jquery.js
      create  public/javascripts/jquery.min.js
     copying  jQuery UJS adapter (0e7426)
      remove  public/javascripts/rails.js
      create  public/javascripts/jquery_ujs.js

```

#### ４．Webサーバ再起動

念のためWebサーバを再起動します。
