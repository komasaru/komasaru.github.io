少し前に、Windows XP マシンに UNIX 用環境の [Cygwin](http://cygwin.com/ "Cygwin") をインストールしたり、Ruby 環境を構築したりしました。

- [Windows - Cygwin インストール！]({{site.baseurl}}/2011/12/02/02002001/ "Windows - Cygwin インストール！")
- [Cygwin - Ruby をソースからインストール！]({{site.baseurl}}/2011/12/04/04002057/ "Cygwin - Ruby をソースからインストール！")

次に、Cygwin 上に高機能テキストエディタ の Vim をインストールしようと考えました。
通常なら、[Cygwin](http://cygwin.com/ "Cygwin") の setup.exe を実行して Vim をインストールしたり、ソースを用意して自前でビルドしたりします。
しかし、折角 Windows 上に GUI 版 Vim の [gVim](http://www.kaoriya.net/ "gVim") をインストールしているのだから、[Cygwin](http://cygwin.com/ "Cygwin") 上からその [gVim](http://www.kaoriya.net/ "gVim") を使用できるようにした方が今後使用し続けていく上で有用ではないかと考えました。

<!--more-->案の定、Web で調べてみるとその方法がいくつか見つかりましたので、まとめました。

以下が手順です。

#### １．既導入の Vim をアンインストール

仮に Vim を既にインストール済みでしたら、不要なのでアンインストールします。

#### ２．シェルスクリプトの作成

Windows 用 GUI 版 Vim の [gVim](http://www.kaoriya.net/ "gVim") 起動用のシェルスクリプトを以下のように作成します。
【ファイル名：/bin/vi】

``` bash 
#!/bin/bash
"C:/Program Files/vim73-kaoriya-win32/gvim"  $(cygpath -aw $*) &
```

※"C:/Program Files/vim73-kaoriya-win32/gvim" は gvim.exe をフルパスで指定。

もしくは、Windows 側の環境変数 PATH に gvim.exe までのパスを指定しておいて、シェルスクリプトを以下のようにしてもよい。

``` bash 
#!/bin/bash
gvim  $(cygpath -aw $*) &
```

#### ３．確認

[Cygwin](http://cygwin.com/ "Cygwin") 側で、

``` bash 
$ vi test
```

などと実行すると、[gVim](http://www.kaoriya.net/ "gVim") が立ち上がります。
