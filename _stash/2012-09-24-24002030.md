Scientific Linux や CentOS のデスクトップ環境ではデフォルトでは Flash Player がインストールされていないので、ブラウザで動画が視聴できません。
（実際は OS に Flash Player をインストールするのではなく、ブラウザにインストールするものですが）

そこで、Flash Player をインストールしてみました。

記録として残しておきます。

<!--more-->

### 作業記録

#### 0. 前提条件

- 作業環境は Scientific Linux 6.3(64) Gnome を想定。
  (CentOS + Gnome 環境でも同じ。おそらく KDE 等も同じでしょう。)

#### 1. Adobe YUM リポジトリ RPM パッケージのインストール

以下のようにして Adobe YUM リポジトリ RPM パッケージをインストールする。

``` text
# rpm -ivh http://linuxdownload.adobe.com/adobe-release/adobe-release-x86_64-1.0-1.noarch.rpm
# rpm --import /etc/pki/rpm-gpg/RPM-GPG-KEY-adobe-linux
```

32bit 環境の場合は `x86_64` の部分を `i386` にすればよい。

#### 2. 各種パッケージのインストール

flash-plugin, nspluginwrapper, alsa-plugins-pulseaudio, libcurl のパッケージをインストールする。

``` text
# yum install flash-plugin nspluginwrapper alsa-plugins-pulseaudio libcurl
```

#### 注意事項

上記では 64bit 版をインストールしていますが、Firefox は 64bit 環境でも 32bit 版をインストールしているはずです。
64bit 版の Flash Player をインストールしただけでは、Firefox では Flash Player は機能しません。
Firefox のために 32bit 版の Flash Player をインストール必要があります。

#### その他

- Firefox に直接プラグインとしてインストールする方法もあります。

- [Flash プラグインをインストールする | 操作方法 | Firefox ヘルプ](http://support.mozilla.org/ja/kb/install-flash-plugin-view-videos-animations-games "Flash プラグインをインストールする | 操作方法 | Firefox ヘルプ")

- Firefox で Flash Player がクラッシュする場合は、Flash Player をバージョンダウンするとよいかもしれません。
当方は、11.2 から 10.3 にバージョンダウンしました。
- Google Chrome では、デフォルトで導入済みの PepperFlash 版 Flash Player から Adobe Flash Player に変更するとクラッシュするかもしれません。
デフォルトの PepperFlash 版 Flash Player を使用した方がよいかもしれません。

---

一応、Firefox, Google Chrome で Flash コンテンツの視聴が可能になりました。

しかし、Google Chrome でデフォルトで導入済みの Shockwave Flash Player を有効にして使用してたところ、You Tube 動画で冒頭に広告が挿入されるケースで Flash Player がクラッシュする現象が発生していました。
おそらく「広告」とか「スキップ」の日本語が影響しているように感じます。
あれこれ弄ったせいか、クラッシュせずに広告の日本語が文字化けする状態にもなることがあります。（※要精査）

また、Adobe Flash Player は Linux 版での開発は 11.2 をもって終了するようです。
PepperFlash(Google Chrome) 向けには提供するようなので、今後は Google Chrome に絞った使い方に変更しようかとも考えている次第です。

※"Pepper(またはPPAPI)" とは Google と Adobe が共同開発したブラウザに API 形式で組み込む Flash Player のことです。

以上。
