まいど、どうも。

メインマシンの重要なデータなどは日々バックアップを取ってます。

でも、もしもWindows自体が起動しなくなった時なんかは、Windowsやらいろんなソフトやらを１からインストールし直さなきゃいけないんで、それは非常にうんざりです。

そこで、時々パーティション丸ごとバックアップを取るようにしてます。

（使うのはKnoppixというLinuxについているPartitionImageというソフトです）

これだと仮にハードディスクが壊れてWindowsが起動しなくなっても、別のハードディスクへリストアすれば、最低限最後にバックアップを取っていた段階へ即戻れるはずです。

（実際にリストアしたことはまだないけど。。。できなかったりして。。。。）

久々に今日やっておきました。

パーティションバックアップするたびにいろんなサイトで調べてたんで、今回は自分の今後のために記録として残しておくことにしました。

### 作業手順

#### １．デフラグ

断片化が多いとバックアップに失敗することがそうです。
僕はいつも「すっきりデフラグ」というフリーソフトを使用してます。

#### ２．Knoppix起動

ＰＣをＣＤから起動し直し。

![01_knoppix]({{site.baseurl}}/images/2009/01/01_knoppix.jpg "01_knoppix")

（Knoppix５．１．１（CD版）を使用。
Knoppix はネット上からダウンロードしてＣＤに焼いてます）
（注）上の画像にうっすら写っているのは霊ではありません。
（僕です。ご安心を。。。）

#### ３．コンソール画面を起動

rootユーザになる。

``` text
knoppix@Knoppix:~$ su -
root@Knoppix:~#

```

#### ４．バックアップ用ディレクトリの作成

``` text
root@Knoppix:~# mkdir /mnt/backup

```

#### ５．作成したバックアップディテクトリをマウント

僕はいつも玄箱に構築しているsambaサーバへマウントしてます。

``` text
root@Knoppix:~# mount -t smbfs -o username=sambaユーザ名 //sambaのマシン名またはIPアドレス/ディレクトリ名 /mnt/backup

```

passwordを聞かれたら、sambaユーザのパスワードを入力。

#### ６．PartitionImageの起動

``` text
root@Knoppix:~# partimage

```

![02_partiamge]({{site.baseurl}}/images/2009/01/02_partiamge.jpg "02_partiamge")

#### ７．保存元・保存先等の設定

![03_partiamge]({{site.baseurl}}/images/2009/01/03_partiamge.jpg "03_partiamge")

今回はＣドライブをバックアップするので、
[ Partition to save/restore ] で “sda1” を選択。
作成したbackupディレクトリに BAK-C という名前でバックアップするので、
[ Image file to create/use ] に “/mnt/backup/BAK-C” と入力。
その他はそのまま。
[ Ｆ５ ] で次へ。

#### ８．オプションの設定

![04_partiamge]({{site.baseurl}}/images/2009/01/04_partiamge.jpg "04_partiamge")

僕は初期設定のままでやってます。
ファイルは２GBずつ分割されます。
ちなみにLinuxでは扱えるファイルのサイズが２GBだとか。。。
（samba では２GB以上のファイルも保存できるけど）

#### ９．圧縮ファイルの説明を入力

![05_partiamge]({{site.baseurl}}/images/2009/01/05_partiamge.jpg "05_partiamge")

別に入力しなくても大丈夫のようです。

#### １０．警告の確認

![06_partiamge]({{site.baseurl}}/images/2009/01/06_partiamge.jpg "06_partiamge")

partimage ではNTFS形式のファイルシステムは正式サポートされてないようですが、
[ OK ] で大丈夫です。

#### １１．情報の確認

![07_partiamge]({{site.baseurl}}/images/2009/01/07_partiamge.jpg "07_partiamge")

確認画面です。
確認後、[ OK ]。

#### １２．バックアップが開始されます

![08_partiamge]({{site.baseurl}}/images/2009/01/08_partiamge.jpg "08_partiamge")

当方の環境では終わるまで５０分くらい。
（バックアップ元が１２．５GBの場合）

#### １３．完了（成功）

![09_partiamge]({{site.baseurl}}/images/2009/01/09_partiamge.jpg "09_partiamge")

[ OK ] 押下。

#### １４．バックアップファイルの確認

![10_result]({{site.baseurl}}/images/2009/01/10_result.jpg "10_result")

sambaサーバに２GBに分割されたバックアップファイルができている。
